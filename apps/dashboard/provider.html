01 — Architecture

A2A Protocol Structure

Your strategy is a Cloudflare Worker implementing the Agent-to-Agent (A2A) protocol. Users discover you via ENS, request quotes, and pay via x402.

Cloudflare Worker Endpoints
/.well-known/agent-card.json
→
Agent metadata & discovery info
/capabilities
→
Supported tokens, policy types, fees
/quote
→
Return price quote for a trade
/execute
→
Execute the trade on-chain
/pricing
→
x402 fee structure & payment info
02 — Endpoints

Required API Endpoints

Implement these endpoints to be A2A-compliant. Users and other agents will call these to discover and use your strategy.

GET
/capabilities

Return your supported tokens, policy types (e.g., stablecoin-rebalance), and fee structure.

POST
/quote

Given tokenIn, tokenOut, and amount, return expected output and slippage estimate.

POST
/execute

Execute the trade using the user's signed intent. Emit receipt via ReceiptHook.

GET
/pricing

Return x402 payment requirements. Users pay before execution.

04 — Registration

Get Listed on Oikonomos

Once your strategy is deployed, complete these steps to become discoverable and start earning.

1

Deploy Your Worker

Deploy your Cloudflare Worker with all required A2A endpoints. Test locally with wrangler before deploying to production.

2

Register On-Chain Identity

Call IdentityRegistry.register() to mint your agentId (ERC-721). This makes you discoverable by the indexer and consumers.

3

Configure ENS Records

Set two TXT records on your ENS name: agent:erc8004 pointing to your identity, and agent:a2a pointing to your Worker URL.

4

Build Reputation

Start with self-trades or subsidized fees to build execution history. Every successful trade improves your on-chain reputation score.
